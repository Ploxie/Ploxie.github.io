Nighlands

The final game made at The Game Assembly

Summary

As the final game project of our education we got to choose what genre we wanted to make. So we decided to make a top-down game with exploration, slower paced combat and simpler graphics design.
We wanted the game to have replay value, so we went for procedurally generated levels and difficulty.

Details

  • Genre: Top-down hack-n-slash
  • Development time: 5 Weeks, full time
  • Language: C++
  • 3rd party tools:
    • Unity
    • Wwise
    • PhysX
  • Team: 11 Members
    • 6 Programmers
    • 2 Level Designers
    • 3 Artists

Level generation

We wanted the levels to be easy to traverse by being linear with a clear idea of where to go, with some deviation for exploration.
The levels had to have a number of key-points like story pieces and bosses.

The environment was built using a variety of 'modules' that were tilable, allowing for seamless connections.
Since each module had different heights, they had to be placed so that every module was accessable, so that the player could finish the level. This was achieved using Perlin-noise.
On generation, the path were traced from start to finish, and certain 'point of interest'-modules were placed along the path. This made sure that, for example, the Boss-levels were guaranteed to have a Boss-module.

Every level also had a 'Difficulty'-value, that increased for every level. This value determined how many hostile creatures would be on the map. Let's say that the first level had a difficulty-value = 1, then after the generation of the terrain, it would spawn a creature worth 1 'difficulty point'. While if the level would have a difficulty-value of for example 3, then the level could spawn with either 3 low-difficulty creatures or 1 high-difficulty creature.
This meant that the further you progressed, the more difficult the levels were.
